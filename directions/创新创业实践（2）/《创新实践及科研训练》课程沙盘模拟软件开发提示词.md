# 《创新实践及科研训练》课程沙盘模拟软件开发提示词

一、软件核心定位与目标

1.  服务场景：适配《创新实践及科研训练》必修课教学，适用专业为智能制造（机械工程）、机械电子工程、车辆工程、工业工程、机器人工程，用于8学时集中模拟企业运营教学环节。

2.  核心目标：构建用友ERP沙盘模拟系统，还原制造企业全流程运营场景，支持学生以小组为单位分工协作，完成企业战略制定、生产运营、市场开拓、财务管理等全流程决策，同时满足教师对模拟过程记录、数据统计及考核评价的需求。

3.  核心原则：严格遵循课程既定运营规则，确保数据计算精准、流程逻辑严谨，所有操作及结果需可追溯，适配教学场景下的便捷操作与高效管控。

二、核心功能模块要求

模块1：系统基础配置模块

1.  角色管理：支持小组分工配置，预设5类核心角色及对应操作权限，角色职责严格如下：

（1）总经理：可查看企业全维度数据（财务、生产、市场、库存），制定发展战略、确定经营指标、制定业务策略、进行全面预算管理、协调管理团队、分析企业绩效、开展业绩考评及管理授权总结。

（2）财务主管：负责日常财务记账登账、模拟向税务部门报税、生成财务报表（资产负债表、利润表）、管理日常现金、制定融资策略、控制成本费用、调度资金与管理风险、通过财务分析辅助决策。

（3）营销主管：开展市场调查分析、制定市场进入与品种发展策略、投放广告、制定销售计划、参与订单谈判与签订、控制订单交付流程、管理应收款、分析销售绩效。

（4）生产主管：负责产品研发管理、管理体系认证、固定资产投资、编制生产计划、平衡生产能力、管理生产车间、保证产品质量、管理成品库存及产品外协。

（5）供应主管：编制采购计划、与供应商谈判签约、监控采购过程、验收到货、管理仓储、决策采购支付、协调财务部与生产部。

2.  小组管理：支持创建多个竞争小组，每个小组对应1家模拟企业，系统可记录各小组运营数据并进行横向对比。

3.  时间维度管控：以“年-季度”为核心时间单位，支持按季度推进运营流程，自动记录当前运营周期，明确年度/季度关键操作节点（如年末贷款、年末缴税、年初销售会议等）。

模块2：模拟企业初始状态配置模块

系统启动时自动为各企业加载固定初始状态，数据不可修改，具体参数如下：

1.  生产中心：拥有大厂房1个（价值40M，可容纳6条生产线），初始配备全自动生产线1条；P1在制品8M（4个），生产线设备总价值13M。

2.  物流中心：R1原料3M（3个），P1成品6M（3个）；R1、R2原材料库，R3、R4原材料库为空；无在途原料及未完成原料订单。

3.  财务中心：现金20M，应收账款15M；长期贷款40M，无短期贷款、应付款及应交税；资产负债表初始数据：流动资产合计52M（现金20+应收款15+在制品8+成品6+原料3），固定资产合计53M（土地建筑40+机器设备13），总资产105M；负债合计41M（长期负债40），所有者权益合计64M（股东资本50+利润留存11+年度净利3），负债+权益105M。

4.  营销与规划中心：已取得P1生产资格，拥有本地市场准入权；未取得P2、P3、P4生产资格，未开展ISO9000、ISO14000认证，未开拓区域、国内、亚洲、国际市场。

模块3：全流程运营模拟模块（核心规则严格遵循以下要求）

1.  市场开拓与准入

（1）市场划分：包含本地（初始已准入）、区域、国内、亚洲、国际5类市场，新市场开拓需满足费用与时间要求：区域市场1M/1年、国内市场2M/2年、亚洲市场3M/3年、国际市场4M/4年。

（2）开拓规则：市场开拓投资按年度支付，每个市场每年最多投资1M，不可加速，可中断；全部投资完成后获得对应市场准入证，方可参与该市场竞单；已准入市场每年需最少投入1M维持，否则自动放弃。

2.  销售会议与订单争取

（1）选单顺序：优先由上年该市场订单价值最高者为市场领导者，最先选单；其次按单一产品广告投入量排序；若单一产品广告投入相同，按该市场全部产品广告投入量排序；若仍相同，按上年订单销售额排序；最后仍相同则通过招标方式确定。

（2）操作要求：每年初触发销售会议，系统展示当前市场需求（产品类型、数量、单价、账期等），各小组营销主管提交广告投入方案后，系统自动按规则排序，支持小组依次选择订单，订单信息需包含产品类型、数量、总额、账期等核心要素。

（3）特殊说明：市场产品数量有限，广告投入不保证必然获得订单。

3.  厂房与生产线管理

（1）厂房规则：提供大、小两种厂房，参数为：大厂房买价40M、租金5M/年、售价40M（计入4Q应收款）、容量6条生产线；小厂房买价30M、租金3M/年、售价30M（计入4Q应收款）、容量4条生产线；年底决策厂房购买/租赁，购买后计入厂房价值，不提折旧。初始配置大、小厂房各1个，每个厂房带1条全自动生产线。

（2）生产线规则：支持4类生产线，参数如下表：

|生产线类型|购买价格|安装周期|生产周期|转产周期|转产费用|维护费用|出售残值|
|---|---|---|---|---|---|---|---|
|手工线|5M|无|3Q|无|无|1M/年|1M|
|半自动|8M|2Q|2Q|1Q|1M|1M/年|2M|
|全自动|16M|4Q|1Q|2Q|4M|1M/年|4M|
|柔性线|24M|4Q|1Q|无|无|1M/年|6M|
（3）生产线补充规则：所有生产线可生产任意产品，加工费均为1M/产品；购买时按安装周期平均支付投资，全部到位下一季度可生产；转产需完成周期及费用支付，最后一笔支付到期下一季度可更换产品；当年在建或出售的生产线免维护费；出售时，净值<残值则净值转现金，净值>残值则残值转现金，差额计入综合费用；折旧按净值1/3取整计算，当年建成不提折旧，净值<3M时每年提1M。

4.  采购与生产管理

（1）采购规则：R1、R2原料订购需提前1季度下单，R3、R4需提前2季度；按上季度订单接收原料入库，同步完成付款或计入应付款；系统需记录原料订单、在途原料及库存数据。

（2）生产规则：生产前需确认原料齐全（缺料则停工待料），按产品结构提交生产计划，支付加工费后启动生产；每条生产线不可同时生产两种产品；本次运营仅支持P1、P2产品，产品结构为：P1=R1+1M加工费，P2=R1+R2+1M加工费。

5.  产品研发

（1）研发要求：P2研发周期6Q、投资6M；P3研发周期6Q、投资12M；P4研发周期6Q、投资18M；本次运营仅开放P2研发。

（2）研发规则：投资可按季度平均支付或延期，资金短缺可中断，全部完成后获得对应产品生产资格；研发投资计入综合费用。

6.  ISO认证

（1）认证参数：ISO9000需≥2年、1M/年投资；ISO14000需≥3年、1M/年投资。

（2）规则：两项认证可同时或延期进行，投资完成后获得资格证，投资计入综合费用。

7.  融资与资金管理

（1）贷款规则：仅支持长期贷款与短期贷款，无高利贷；长期贷款每年年末申请，额度为权益2倍，年息10%，期限3年，每次20M，最多40M，年底付息、到期还本；短期贷款每季度初申请，额度为权益2倍，年息5%，期限1年，1月、6月放贷，每次20M，最多40M，到期一次还本付息；贷款金额为20的倍数。

（2）资金贴现：有应收款时可随时进行，金额为7的倍数，按1:6贴息（7M应收款贴现需支付1M费用，实际获得6M现金）。

（3）禁止规则：不允许民间借贷、层层转包等违法行为。

8.  费用、折旧与税金

（1）综合费用：包含行政管理费（年底收1M）、市场开拓费、产品研发费、ISO认证费、广告费、生产线转产费、设备维护费、厂房租金等，实时计入对应年度费用。

（2）折旧：按余额递减法，每年按生产线净值1/3取整计算，当年建成不提折旧，净值<3M时每年提1M。

（3）税金：每年所得税计入应付税金，下年初交纳。

模块4：数据统计与报表模块

1.  实时数据记录：自动记录各小组每季度/年度的核心运营数据，包括现金流量、原料库存、成品库存、生产线状态、订单完成情况、广告投入、市场开拓进度、研发进度、贷款与还款情况、费用支出等。

2.  报表生成：自动生成年度利润表、资产负债表，报表数据需与运营操作严格对应，支持导出与打印。

3.  小组对比：支持按年度展示各小组的关键指标对比（如净利润、市场份额、资产总额等），为教师考核提供数据支撑。

模块5：教学考核辅助模块

1.  过程记录：记录各小组的运营控制情况、角色操作轨迹，支持教师查看任意小组的运营过程。

2.  成绩评定支撑：按课程考核规则（总评成绩=平时表现20%+模拟企业运营70%+实践报告10%），支持教师录入平时表现分数，系统自动统计模拟企业运营相关数据（小组排名、运营指标完成情况等），生成初步考核结果。

3.  规则管控：系统内置运营规则校验机制，对违规操作（如超额度贷款、未达标开拓市场竞单、缺料生产等）进行提示并限制操作，确保模拟符合课程要求。

三、特殊运营规则补充（需严格适配）

1.  时间推进：第一年为建设周期，无产品产出；区域市场第二年可准入，国内市场第三年可准入。

2.  选单简化：本次运营广告投入直接决定选单顺序，无需考虑其他附加排序条件。

3.  初始配置：系统启动时自动为各小组分配大、小厂房各1个，每个厂房配套1条全自动生产线，无需额外购买。

四、技术与体验要求

1.  操作便捷性：界面布局清晰，角色操作权限区分明确，关键操作流程有引导提示，支持快速定位核心功能（如订单选择、生产计划提交、贷款申请等）。

2.  数据准确性：所有财务计算、周期统计、规则校验需精准无误，确保运营结果真实反映决策效果。

3.  稳定性：支持多小组同时在线操作，无卡顿、数据丢失问题；关键操作（如订单确认、贷款申请）需有数据备份。

4.  兼容性：支持主流浏览器访问，适配电脑端教学场景使用。
